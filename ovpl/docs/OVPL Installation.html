1 OVPL Installation Process

1.1 Setting up the proxy if required.

  • Updates may fail if the proxy is set to the network
  • Contact the admin to know the proxy and set it to the $httpproxy variable

1.2 Update the package manager

  • Update the package manager by running the command,

yum update -y

1.3 Install Git on your system

  • Install git and get familiar with basic commands

1.4 Cloning The Lab Integration Kit (LIK)

  • Clone the directory "ovpl" from Github LIK repository.

1.5 Installing OpenVZ

  • The first step is to install openVZ
  • Go to "ovpl" directory and in "scripts" directory, You will find a script named, "openVZinstall.sh" which installs OpenVZ on Your CentOS.
  • To run this script, go to that directory in terminal and run the following command: 
sudo sh openVZ_install.sh
  • Restart the system and login with openVZ kernel

1.6 Installing OVPL Dependencies

  • Before starting the instalaltion, we need to install all the dependencies
  • Go to OVPL directory and in "dependencies" directory, You will find a script named, "dependencies.sh" which installs all the dependencies required to install OVPL.
  • To run this script, go to that directory in terminal and run the following command, 
sudo sh dependencies.sh

1.7 Installing OVPL

  • Go to

../ovpl/src/src/adapters/

and run AdapterServer.py as a background process. 

python AdapterServer.py &
  • Now, go to ovpl/src/src and run ControllerServer.py. Now You are ready to deploy the lab.
  • Go to browser and type, "127.0.0.1:8080", you will see a test page.
  • Enter lab id and URL and submit it.
  • You can see the deployment status of the lab in the terminal. The IP address of the container is prompted in the terminal where the lab is being deployed.
  • Lab can be hence used from the given IP.
